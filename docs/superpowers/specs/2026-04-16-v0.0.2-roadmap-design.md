# slopgate v0.0.2 Roadmap Design

**Date:** 2026-04-16
**Status:** Design approved
**Target:** Community-adoptable AI-slop pre-commit gate with configurable rules and cross-language coverage

## Problem

slopgate v0.0.1 ships 5 rules with hard-coded severity and Go+JS/TS+Python coverage. To be useful as a community tool, it needs:

1. **Configurability** ‚Äî teams must be able to downgrade block‚Üíwarn or suppress rules per-path without patching source
2. **Broader detection** ‚Äî 4 more AI-slop patterns that v0.0.1 doesn't catch (error swallowing, tautological assertions, stub bodies, unused imports), plus expansions to v0.0.1 rules
3. **Community readiness** ‚Äî rule catalog, config docs, contribution guide

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Config format | TOML (`.slopgate.toml`) | Standard in Go ecosystem, supports comments, single well-maintained parser dep |
| New dependency | `github.com/BurntSushi/toml` | Only dep beyond stdlib; zero transitive deps, MIT, actively maintained |
| Milestone structure | alpha (config) ‚Üí beta (Go rules) ‚Üí rc (cross-lang + polish) | Config ships first so every new rule is configurable from day one |
| Rule quality bar | Zero false positives on real repos (proven via regression tests) | User chose precision: every finding must be gold |
| Config scope | Severity override + per-rule ignore + per-rule path ignores | Middle ground: not locked to hard-coded, not full inline `noslopgate` |

## Milestones

### v0.0.2-alpha: Config Foundation

1. Add `github.com/BurntSushi/toml` dependency

3. Wire config into CLI: `--config` flag, auto-discovery from repo root

8. Tag `v0.0.2-alpha`

### v0.0.2-beta: Go-Native Rules

Each rule ships with:
- Implementation in `pkg/rules/slpXXX.go`
- Regression tests in `pkg/rules/slpXXX_test.go`
- Self-check test covering slopgate's own codebase

Rules:

| ID | Name | Severity | Catches |
|---|---|---|---|
| SLP002 | Tautological assertion | block | Go: `assert.Equal(t, x, x)`, `if true {}` in test bodies |
| SLP003-Go | Empty error handler (Go) | warn | `if err != nil { }` with no statement in the block, or `if err != nil { return nil }` with no logging |
| SLP006 | Panic/throw stub body | block | Go: `panic("not implemented")`, `panic(fmt.Sprintf("TODO"))` |
| SLP007-Go | Unused new import (Go) | warn | Import added in diff but identifier never referenced in any added line |
| SLP008-Go | Log-and-continue (Go) | warn | `log.Printf(...); return nil` or `slog.Error(...); return` ‚Äî error logged but silently returned |
| SLP010 | Incremental no-assertion test | info | Pre-existing Go test function where added lines contain no assertion (SLP001 expansion) |

### v0.0.2-rc: Cross-Language Rules + Community Polish

Add cross-language patterns to rules that have them:

- **SLP002** (tautological assertion) ‚Äî add JS/TS (`expect(x).toBe(x)`, `assert.strictEqual(x, x)`) and Python (`self.assertEqual(a, a)`, `assert a == a`)
- **SLP003** (empty catch/except) ‚Äî add JS/TS (`catch (e) {}`, `catch {}`) and Python (`except: pass`, `except Exception: pass`)
- **SLP006** (stub bodies) ‚Äî add JS/TS (`throw new Error("not implemented")`, `throw new Error("TODO")`) and Python (`raise NotImplementedError`)
- **SLP007** (unused imports) ‚Äî add JS/TS (`import { X } from 'y'` where `X` is never used in added lines)
- **SLP008** (log-and-continue) ‚Äî add JS/TS (`console.error(...); return`) and Python (`logger.error(...); return None`)
- **SLP009** (env-var drift) ‚Äî Go and JS/TS: `os.Getenv("X")` or `process.env.X` added in diff with no matching `os.Setenv` reference in the same diff. Does NOT check `.env` file contents (that would require unrelated-file cross-referencing beyond diff-only scope); only flags lookups of env vars that aren't set anywhere in the same change
- **SLP011** (assert-only test body) ‚Äî Go, JS/TS, Python: test function body is entirely assertion calls with no meaningful logic

README overhaul:
- Full rule catalog table with IDs, descriptions, languages, severities
- Config reference with examples
- Contribution guide (adding a new rule)
- Installation via `go install` and pre-commit hook setup

Tag `v0.0.2`.

## Config Format

`.slopgate.toml` ‚Äî auto-discovered from repo root or specified via `--config PATH`:

```toml
# Override rule severity. Values: "block", "warn", "info", "off".
# "off" is equivalent to ignore = true but more explicit.
[rules.SLP012]
severity = "warn"

[rules.SLP014]
severity = "info"

# Ignore a rule entirely.
[rules.SLP009]
ignore = true

# Per-rule path ignores (additive to .slopgateignore).
[rules.SLP003]
ignore_paths = ["vendor/**", "**/migrations/**"]
```

### Discovery order

1. `--config PATH` if provided
2. Walk up from `-C` dir to repo root looking for `.slopgate.toml`
3. No config found ‚Üí all rules use `DefaultSeverity()`

### Config struct

```go
type Config struct {
    Rules map[string]RuleConfig `toml:"rules"`
}

type RuleConfig struct {
    Severity    string   `toml:"severity"`     // "block", "warn", "info", "off"
    Ignore      bool     `toml:"ignore"`       // equivalent to severity = "off"
    IgnorePaths []string `toml:"ignore_paths"`  // glob patterns, additive to .slopgateignore
}
```

The `Severity` field maps to `rules.Severity` values. `"off"` and `Ignore = true` both exclude the rule from the run. `IgnorePaths` is merged with `.slopgateignore` patterns at the per-rule level ‚Äî a file matched by `SLP003.ignore_paths` only skips SLP003, not all rules.

## ID Numbering

SLP002‚ÄìSLP011 skips SLP004 (reserved from a v0.0.1 cut rule that was too noisy ‚Äî SLP001 tests that add only comments). Existing IDs (SLP001, SLP005, SLP012‚ÄìSLP014) are stable and will not be renumbered.

## Architecture Impact

### New package: `pkg/config`

- `config.go` ‚Äî `Config`, `RuleConfig` structs
- `config.go` ‚Äî `Load(path string) (*Config, error)` ‚Äî parse TOML
- `config.go` ‚Äî `Discover(repoDir string) (string, error)` ‚Äî walk up to repo root

### Modified: `pkg/rules/registry.go`

- `Run()` accepts a `*config.Config` (or nil for defaults)
- Per-rule severity override applied before findings are returned
- Per-rule ignore causes the rule to be skipped entirely
- Per-rule `ignore_paths` filters files before the rule's `Check()` runs

### Modified: `cmd/slopgate/main.go`

- Add `--config` and `--list-rules` flags
- Load config after diff parsing, pass to `Registry.Run()`
- `--list-rules` prints and exits

### New: One file per rule

- `pkg/rules/slp002.go` + `slp002_test.go`
- `pkg/rules/slp003.go` + `slp003_test.go`
- `pkg/rules/slp006.go` + `slp006_test.go`
- `pkg/rules/slp007.go` + `slp007_test.go`
- `pkg/rules/slp008.go` + `slp008_test.go`
- `pkg/rules/slp009.go` + `slp009_test.go`
- `pkg/rules/slp010.go` + `slp010_test.go`
- `pkg/rules/slp011.go` + `slp011_test.go`

### Modified: `go.mod`

- Add `github.com/BurntSushi/toml` dependency

## Out of Scope for v0.0.2

- Inline `// noslopgate` suppression comments (can be added in v0.0.3)
- SARIF output format (can be added in v0.0.3)
- GitHub Actions integration (separate repo/docker image, future)
- Reviewdog integration (future)
- Plugin/extension system for custom rules (future)
- SLP004 (comment-only test additions ‚Äî too noisy, remains reserved)
