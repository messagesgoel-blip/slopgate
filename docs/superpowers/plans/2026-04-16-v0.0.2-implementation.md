```go
package config

import (
	"os"
	"path/filepath"

	"github.com/BurntSushi/toml"
)

// Config represents the .slopgate.toml structure.
type Config struct {
	Rules map[string]RuleConfig `toml:"rules"`
}

// RuleConfig configures a single rule.
type RuleConfig struct {
	Severity    string   `toml:"severity"`     // "block", "warn", "info", "off"
	Ignore     bool     `toml:"ignore"`       // true to disable the rule
	IgnorePaths []string `toml:"ignore_paths"`  // glob patterns, per-rule file ignores
}

// Load parses a TOML config file.
func Load(path string) (*Config, error) {
	cfg := &Config{}
	_, err := toml.DecodeFile(path, cfg)
	if err != nil {
		return nil, err
	}
	return cfg, nil
}

// Discover walks up from dir to find .slopgate.toml.
// Returns empty string if not found.
func Discover(dir string) (string, error) {
	dir, err := filepath.Abs(dir)
	if err != nil {
		return "", err
	}
	for {
		tomlPath := filepath.Join(dir, ".slopgate.toml")
		if _, err := os.Stat(tomlPath); err == nil {
			return tomlPath, nil
		}
		parent := filepath.Dir(dir)
		if parent == dir {
			break // reached root
		}
		dir = parent
	}
	return "", nil
}
```

### Task 6: SLP002 — Tautological Assertion

**Catches:** Go: `assert.Equal(t, x, x)`, `if true {}` in test bodies. Also JS/TS/Python in RC.

**Test cases:**
- `assert.Equal(t, x, x)` → finding
- `assert.Equal(t, 1, 1)` → finding
- `assert.True(t, true)` → finding
- `assert.Equal(t, got, got)` → finding
- `assert.Equal(t, got, got)` where got is from function call (NOT a finding — depends on runtime)

**Implementation notes:**
- Use regex: `assert\.Equal\s*\([^,]+,\s*([^,]+),\s*\1\s*\)` for identical args
- Or simpler: flag `assert.Equal(t, x, x)` pattern only (exact same identifier)
- SLP002 is warn-by-default per spec

---

### Task 7: SLP003 — Empty Catch/Except

**Catches:** Go: `if err != nil { }` with no statement. JS/TS: `catch (e) {}`. Python: `except: pass`.

**Test cases:**
- `if err != nil { }` → finding
- `if err != nil { return nil }` with no log (NOT logged) → finding
- `if err != nil { log.Println(err); return nil }` → NOT a finding (logged)
- `catch (e) {}` → finding
- `except Exception: pass` → finding

**Implementation notes:**
- Go: Check for `if err != nil {` pattern, then scan block for any non-comment statement
- If block is empty or only contains `return`/`return nil` without logging → finding
- Check for `log.`, `slog.`, `fmt.` inside the block — if found, NOT a finding

---

### Task 8: SLP006 — Panic/Throw Stub Body

**Catches:** Go: `panic("not implemented")`, `panic(fmt.Sprintf("TODO"))`. JS/TS: `throw new Error("not implemented")`. Python: `raise NotImplementedError`.

**Test cases:**
- `panic("not implemented")` → finding
- `panic("TODO")` → finding
- `panic(fmt.Sprintf("fixme"))` → finding
- `throw new Error("not implemented")` → finding
- `raise NotImplementedError` → finding
- `panic(err)` (dynamic) → NOT a finding

**Implementation notes:**
- Match literal string args containing "not implemented", "TODO", "fixme", "unimplemented"
- Case-insensitive

---

### Task 9: SLP007 — Unused New Import

**Catches:** Go: Import added in diff but identifier never referenced in any added line.

**Test cases:**
- `import "foo"` added, `foo.Bar()` in same diff → NOT a finding
- `import "foo"` added, no reference in diff → finding
- `import "foo"` already existed → NOT a finding (only new imports)

**Implementation notes:**
- First collect all imported package names from added `import` lines
- Then scan all added lines for any identifier that could reference that package
- Only fire if import IS NEW (in diff) AND no references in added lines

---

### Task 10: SLP008 — Log-and-Continue

**Catches:** Go: `log.Printf(...); return nil`. JS/TS: `console.error(...); return`. Python: `logger.error(...); return None`.

**Test cases:**
- `log.Printf("err: %v", err); return nil` → finding
- `slog.Error("msg"); return` → finding
- `console.error(err); return` → finding
- `logging.error(err); return None` → finding
- `log.Printf("err: %v", err); return nil; someOtherCode()` → NOT a finding (not pure log-and-return)

**Implementation notes:**
- Match log statement followed by return statement on same or next line
- The log must be BEFORE the return (not after recovery)
- Only flag if the log contains "error", "fail", or similar error semantics

---

### Task 11: SLP009 — Env-Var Drift

**Catches:** Go: `os.Getenv("X")` or JS/TS: `process.env.X` added in diff with no matching `os.Setenv` reference in the same diff.

**Test cases:**
- `os.Getenv("FOO")` added, no `os.Setenv("FOO")` or `os.LookupEnv` in diff → finding
- `os.Getenv("FOO")` added, `os.Setenv("FOO", "bar")` in same diff → NOT a finding
- `os.Getenv("FOO")` was already in file (not in diff) → NOT a finding

**Implementation notes:**
- Only check ADDED lines
- Collect all `os.Getenv("NAME")` calls from added lines
- Collect all `os.Setenv("NAME", ...)` and `os.LookupEnv("NAME")` from added lines
- If lookup exists without set in diff → finding
- Does NOT check .env file contents (out of scope per spec)

---

### Task 12: SLP010 — Incremental No-Assertion Test

**Catches:** Pre-existing Go test function where added lines contain no assertion (SLP001 expansion).

**Test cases:**
- Existing test `TestFoo` with 5 lines, added new line calling `Foo()` with no assertion → finding
- Existing test `TestFoo` with 5 lines, added new line `t.Errorf(...)` → NOT a finding
- Entirely new test function → SLP001 catches this, SLP010 skips

**Implementation notes:**
- Only applies to test files where function ALREADY exists (not new in diff)
- Scan added lines in test bodies — if ANY added line contains assertion, NOT a finding
- Reuses SLP001's hasAssertion() logic

---

### Task 13: SLP011 — Assert-Only Test Body

**Catches:** Test function whose body is entirely assertion calls with no meaningful logic.

**Test cases:**
- `func TestFoo(t *testing.T) { assert.Equal(t, 1, 1) }` → finding (no setup/arrange)
- `func TestFoo(t *testing.T) { if got := Foo(); assert.Equal(t, 1, got) }` → NOT a finding (has arrange)
- Python: `def test_foo(): assert_equal(1, 1)` → finding
- JS: `test('foo', () => { expect(1).toBe(1) })` → finding

**Implementation notes:**
- Scan test body — if ALL lines are assertions (no variable assignments, no function calls except assertions) → finding
- Allow one-level variable assignment for the "arrange" value: `got := Foo()` followed by assertion is OK
